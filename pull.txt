Git 上拉取文件到本地，根据不同的场景有不同的操作方法，下面为你详细介绍

场景一：克隆全新的远程仓库到本地
git clone

场景二：更新已有本地仓库的文件
从 Git 上拉取文件到本地，根据不同的场景有不同的操作方法，下面为你详细介绍：
git pull <远程仓库名称> <分支名称>
常见的远程仓库名称默认为 `origin`，如果你要拉取 `master` 分支的更新，可以使用以下命令：
git pull origin master
如果你的本地分支已经和远程分支建立了跟踪关系
如使用 `git clone` 克隆仓库后，或者使用 `git push -u origin master` 推送过一次），可以直接使用 `git pull` 命令，Git 会自动拉取跟踪分支的更新。

场景三：拉取特定分支或标签
如果你只需要拉取远程仓库的特定分支或标签，可以使用以下方法：
git clone -b <分支名称> <远程仓库地址>
例如，要克隆远程仓库的 `dev` 分支：
git clone -b dev https://github.com/username/repository.git
如果你已经有本地仓库，要切换到特定分支并拉取更新，可以使用以下命令：
git checkout <分支名称>
git pull origin <分支名称>
拉取特定标签
git clone <远程仓库地址>
cd <本地仓库目录>
git checkout <标签名称>
例如，要拉取远程仓库的 `v1.0` 标签：
git clone https://github.com/username/repository.git
cd repository
git checkout v1.0
